Built for the part normal awake apps avoid.

Normal idle-sleep blockers are useful while the Mac is open. Don't Stop also gives lid-closed work a product surface, permission memory, and cleanup path.

Lid mode is explicit, remembered, and reversible.

When you enable lid-closed running, Don't Stop asks once, installs the narrow permission it needs, and avoids repeating the same admin prompt during future toggles.

✓ Clear stateMenu bar text changes from Ready to Awake or Lid so you know what is active.
✓ Scoped controlThe app only asks for what is needed to switch lid mode on and off.
✓ Cleanup behaviorSessions can be timed, stopped manually, or ended when a wrapped command exits.

Why this instead of another keep-awake toggle?

Caffeinated sessions are table stakes. Agent workflows need clearer state, fewer repeated prompts, and a way to attach awake behavior to the job itself.

Questions before you trust it overnight.

The app is intentionally small, local, and visible. You should always know when it is protecting a run.

Will it ask for my password every time?

No. Lid mode is designed to ask once, remember the permission, and switch without repeating the same prompt on future toggles.

Does the display have to stay on?

No. You can keep the Mac awake while still allowing the display to sleep, or keep the screen on when you need visible progress.

Is lid-closed running always safe?

Use it intentionally. Keep the Mac on a hard surface with airflow, preferably on power, and stop the session when the job is done.

Can agents use it directly?

Yes. The helper and the Don't Stop skill let Codex or Claude protect long-running work, check current state, and release protection after completion.
